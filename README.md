# Ecommerce Data Cleaning

A simple e-commerce transaction data cleaning project using `pandas`.

This repository demonstrates how to load a messy e-commerce transactions dataset, clean and normalize key fields, remove duplicates, and export a ready-to-use cleaned CSV file.

## Files

- `DataCleaningTask.ipynb` - Jupyter notebook containing the data loading, cleaning pipeline, validation, and export steps.
- `messy_ecommerce_transactions.csv` - Source dataset with inconsistent formatting, missing values, duplicate transaction IDs, and mixed data types.
- `cleaned_ecommerce_transactions.csv` - Output dataset generated by the notebook after data cleaning.

## What this project does

The notebook reads the messy dataset and applies the following cleaning steps:

- trims whitespace in text fields
- normalizes product names and fills missing values with `Unknown`
- converts prices to numeric values with two decimal precision
- standardizes order dates to `YYYY-MM-DD`
- normalizes payment status values to `Success`, `Pending`, or `Failed`
- removes duplicate transaction records based on `Transaction_ID`
- fills missing prices with the median price and missing payment statuses with `Pending`

## Requirements

- Python 3.8+
- pandas
- numpy

Install dependencies with:

```bash
pip install pandas numpy
```

## How to run

Open `DataCleaningTask.ipynb` in Jupyter Notebook or JupyterLab and execute the notebook cells.

If the file `messy_ecommerce_transactions.csv` is missing, the notebook will generate a small sample dataset automatically.

After running the notebook, the cleaned output is saved to:

- `cleaned_ecommerce_transactions.csv`

## Notes

This project is intended as a data cleaning example for e-commerce transaction logs. The cleaning logic can be extended for larger datasets, additional business rules, or more advanced validation.
